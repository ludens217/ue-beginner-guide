📦 新人入坑 UE 的第一周路线图

▶ Day 1｜完全不懂也没关系

Day 1 的目标只有一个：
👉 不慌、不怕、能在 UE 里点 Play

🧠 先说一句人话

如果你现在看见 UE 编辑器就头大， 不知道什么是关卡、Actor、蓝图—— 完全正常。

UE 是给团队用的工业工具， 今天不要求你“会做东西”，只要求你敢点。

🧩 今天只记住 3 个名字

  • 项目（Project）：一个游戏 = 一个文件夹
  • 关卡（Level）：一个场景
  • Play：运行当前关卡（最重要）

🎮 实际操作

  1. 打开 Unreal Engine
  2. 打开任意一个项目（示例 / 空白都行）
  3. 什么都别改，直接点 ▶ Play

🎉 Day 1 完成。

🏁 Day 1 合格线

你能对自己说一句： “我知道 UE 是干嘛的，我敢点 Play。”

▶ Day 2｜蓝图不是代码（从 0 做一扇自动门）

Day 2 的目标：
👉 看懂蓝图在“干嘛”，并且真的做出一扇能开、能关、不抽搐的门

🧠 先说一句最重要的人话

蓝图不是“写程序”，而是：

  • 什么时候发生事？
  • 发生后要做什么？
  • 这个东西属于谁？

今天你只需要理解一句话：
「门的逻辑，应该写在门自己身上。」

📦 Step 1｜新建一个“门蓝图”

  1. 在 Content Drawer 空白处右键
  2. Blueprint Class → Actor
  3. 命名为 BP_Door
  4. 双击打开

👉 这是你的第一条“UE 习惯”：
会动、会响应的东西，基本都该是 Actor 蓝图。

🧱 Step 2｜给门加组件（不加逻辑）

在 Components 面板里：

  • ➕ Static Mesh（门本体）
  • ➕ Box Collision（触发区域）

📌 注意：
Box 要比门大，代表“人靠近门”

（这里你可以以后放一张：门 + Box 的截图）

⚡ Step 3｜第一个事件：人走进来了

切到 Event Graph：

  1. 选中 Box Collision
  2. 右键 → Add Event → On Component Begin Overlap

这句话的意思是：
“有东西走进这个区域了”

🧩 Step 4｜不要直接动门（新手最容易错）

❌ 错误做法：
On Overlap → Set Location（立刻改位置）

结果通常是：
门瞬移 / 抽搐 / 看起来像坏掉

✅ 正确做法：
用 Timeline 控制“变化过程”

⏱️ Step 5｜Timeline：门为什么不抽搐

  1. 在蓝图中右键 → Add Timeline
  2. 命名为 DoorTimeline
  3. 打开 Timeline
  4. 添加一个 Float Track（0 → 1）

📈 0 = 门关着
📈 1 = 门完全打开

（这里非常适合放一张 Timeline 曲线截图）

🚪 Step 6｜用 Timeline 推动门移动

在 Timeline 的 Update 引脚：

  • 使用 Lerp（Vector）
  • A = 关门位置
  • B = 开门位置
  • Alpha = Timeline 的输出

👉 Timeline 不“控制门”，
它只负责告诉你：
“现在是 0.3 / 0.7 / 1”

🔁 Step 7｜关门怎么办？

再加一个事件：

  1. Box Collision → On Component End Overlap
  2. 连接到 Timeline 的 Reverse

👉 人走近 → Play
👉 人走远 → Reverse

⚠️ 新人 100% 会踩的坑

  • ❌ 用关卡蓝图控制门
  • ❌ 每一帧 Set Location
  • ❌ 门和触发器不是一个 Actor

🏁 Day 2 合格线

你不需要记住所有节点名字，
只要你能说出这句话：

“门自己知道什么时候该开，该怎么动。”

Day 3｜动画 & 交互

  • Timeline 控制移动
  • 音效跟随门的高度变化

Day 4–7｜开始像个 UE 人

  • 拆分蓝图职责
  • 避免关卡蓝图过重
  • 做一个完整的小机关

✨ 这是给「完全不会 UE 的人」准备的路线图。
